# План приложения DiaryTaskManager (MAUI)

## Цель
Быстрый ввод задач и удобное планирование по горизонту: **день / неделя / месяц / позже**.

## Основная идея UX
- **Быстрый ввод (Quick Add)**: одно поле + выбор горизонта (Сегодня/Неделя/Месяц/Входящие) + кнопка "Добавить".
- **Входящие (Inbox)**: всё, что записано "на потом/без даты" и ждёт разбор.
- **План (Plan)**: просмотр по секциям (Сегодня/Неделя/Месяц) + отметка выполнения + перенос между секциями.

## Экраны (MVP)
- **Quick Add**: добавить задачу (текст, горизонт, опционально заметка).
- **Plan**: список задач по горизонтам, перенос, завершение.
- **Inbox**: список входящих, назначение горизонта, удаление.
- **Settings (позже)**: темы, резервное копирование/синхронизация, напоминания.

## Сущности (минимум)
- `TaskItem`: Id, Title, Notes, Bucket, IsDone, CreatedAt, DueDate (опционально).
- `ScheduleBucket`: Inbox, Today, ThisWeek, ThisMonth, Later.

## Хранение данных (этапы)
- **Этап 1 (сейчас)**: in-memory репозиторий (чтобы быстро проверить UX).
- **Этап 2**: локальная БД (SQLite) + миграции.
- **Этап 3 (опционально)**: синхронизация (файл/облако), конфликт-резолв.

## Архитектура (для удобной разработки)
Feature-first + MVVM:
- `Features/*/Views` — страницы
- `Features/*/ViewModels` — VM
- `Core/*` — модели и абстракции
- `Data/*` — реализации репозиториев/хранения
